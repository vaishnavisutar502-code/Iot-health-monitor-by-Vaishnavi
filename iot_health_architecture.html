Layer 1 — Hardware / Sensor Layer
Physical data collection from patient
01 / 05
MAX30102 Sensor
Pulse oximeter & heart rate sensor. Measures SpO₂ (blood oxygen %) and BPM using infrared light absorption.
SpO₂ Reading Heart Rate BPM
Arduino Mega 2560
Microcontroller that reads sensor data over I²C protocol, processes raw values, and sends data to WiFi module.
I²C Bus Serial UART
ESP8266 / ESP32 WiFi
Connected to Arduino via Serial. Receives sensor values and performs HTTP POST to ThingSpeak API every 15–30 seconds.
WiFi 802.11 HTTP POST
📡 Data Flow: MAX30102 → I²C → Arduino Mega → UART → ESP8266 → WiFi → ThingSpeak REST API
📦 Payload: {"field1": <BPM>, "field2": <SpO2>} sent to ThingSpeak channel
HTTP POST (REST API)

🔔 Alerts Panelst.warning / st.error banners for abnormal readings
🚀 Key Streamlit Features Used:
• st.metric() — Live BPM and SpO₂ big number display
• st.plotly_chart() — Interactive historical + real-time line charts
• st.selectbox() — Choose ML model (KNN / LR / Proto Net)
• st.cache_data(ttl=15) — Cache ThingSpeak API calls for 15 seconds
• st.rerun() + time.sleep(15) — Auto-refresh live data loop
• st.sidebar — Navigation, patient info, alert config
Browser renders Streamlit app (localhost:8501)
